前端路由实现

SPA是近几年比较核心的前端应用构建方式
其有许多优点 如用户体验ok
但是也有弊端 比如首屏渲染慢 但这不是这次要讨论的问题

在使用vue-router或者react-router的时候 用起来确实十分的爽快
但是有时候也要知其原理 刨根问底 知其所以然
这样才能保持对技术追求的热情 有敏锐的技术嗅觉

实现前端路由主要依靠浏览器的history API和hash
前者对浏览器的兼容性是有一定要求的 后者的兼容性十分好 所以比如vue-router默认的是采用hash的模式来进行路由管理

但两者实现的方式差不多 只有api上的变化

History API

这基于History API的方式

依靠popstate事件和history的pushState方法来进行路由监听和路由的注册
通过我们的伪路由表(俩a连接) 然后对他们注册事件 先阻止了默认事件 然后将对饮的链接push到history中 就增加了一个浏览记录并跳转
监听路由会发生在前进后退的操作上

Hash

基于hash的方式 和history差不多 代码很像

如果知道了history的方式 那hash的方式也很容易理解了
差别就是修改hash 浏览器会自动添加到历史记录中
另外监听hash的改变 就达到了监听路由的效果
