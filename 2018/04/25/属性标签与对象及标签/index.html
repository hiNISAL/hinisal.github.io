属性标签与对象及标签

在学了解原型链的时候 会触及到很多对对象的操作

对象上的每个属性都有对应的属性标签

属性标签描述了这个对象是否可以被枚举 这个对象的值 或者对象是否可写等

获取一个对象的属性标签很简单通过


Object.getOwnPropertyDescriptor(obj, property)



这个方法就可以返回一个属性标签的描述器

比如：


Object.getOwnPropertyDescriptor({name: “nisal”}, “name”);

返回
Object{value: “nisal”, writable: true, enumerable: true, configurable: true}

他返回了四个标签


  • value 表示属性值

  • writable 表示只读或者可写

  • enumerable 表示是否可以被枚举

  • configurable 表示该属性是否可以被delete或者这些属性标签的值是否可以再被修改


比如我们在用for in遍历对象的时候 如果原型链上的属性的enumerable标签设置成了true 那么就会被遍历到 要是只想遍历到部分原型链上的属性

那么只要修改这个标签就不会被遍历到了

我们可以通过自己创建对象的属性 来设置这些标签


Object.defineProperty使用这个方法来给指定的对象创建属性


var person = {};
Object.defineProperty(person, ‘name’, {
  configurable: true,
  writable: true,
  enumerable: true,
  value: ‘nisal’
});
Object.keys(person); // [name]

使用这个方法创建属性就可以指定他属性标签的值

但是这个方法一次只能定义一个属性

如果想要一次性定义多个 就可以使用

Object.defineProperties(obj, {属性集合}) 这个方法


Object.defineProperties(person, {
  name: {value: ‘nisal’, enumerable: true},
  age:  {value: 14, enumerable: true, writable: true}
});

这样就一次创建了多个属性

【如果在创建的时候没有指定属性标签的值的话 那么值就会设置成false】

另外只要当configurable这个标签的值为true 就可以通过Object.prototype来修改其他的属性标签的值

对象级别的标签


proto　这个标签指向原型　已经了解了

还有两个对象级别的标签


  • class 类型

  • extensible 表示是否可以扩展　对象上的属性是否可以继续增加


这俩没有像原型那么多的内容

class

var toString = Object.prototype.toString;
function getType(o){return toString.call(o).slice(8, -1);}  //call调用toString方法 o作为this传入
toString.call(num); [object Null]
getType(null); “Null”
getType(undefined); “Undefined”
getType(1); “Number”
getType(new Number(1));  “object”
getType(true); “Boolean”

extensible

extensible 表示是否可扩展 对象上的属性是否可以继续添加
var obj = {x:1, y:2};
Object.isExtensible(obj);  返回true
Object.preventExtensions(obj);
Object.isExtensible(obj);  返回false
obj.z = 1;
obj.z;   返回undefined,add new property failed

Object.seal(obj); 禁止修改属性标签和删除属性【configurable: false】
Object.isSealed(obj); 返回true 判断是否被seal

Object.freeze(obj); 在seal的基础上 不可修改值【wtritable: false】
Object.isFrozen(obj); 返回true
对原型链没有影响
