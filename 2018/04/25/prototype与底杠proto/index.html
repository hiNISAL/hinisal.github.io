prototype

重新回顾了一下原型的内容 发现之前还有一点没搞明白的地方

之前说到属性标签proto和属性prototype

之前把这两个之间的关系搞混了

其实是这样

当申明了一个函数 这个函数酒会默认带上一个参数 或者说是属性 这个属性就是prototype

这个prototype又会带上两个参数 一个是constructor 另一个就是 proto

这是在chrome控制台里的样子 其实这两个属性是chrome暴露出来的

那么具体prototype和 proto的关系怎么样呢

function Parent(){
  this.name = ‘nisal’;
}
Parent.prototype.age = 16;
var father = new Parent();

这个函数 是一个构造函数

根据之前有关this的那一内容 一个函数的this 默认指向的是全局对象

但是当他作为构造函数的时候 他的this会指向一个空对象 并且这个对象的原型会指向这个函数的prototype属性

和刚才差不多 申明一个函数Demo 他默认会又一个prototype属性 这个属性是默认包含constructor和proto的对象

当把这个Demo函数作为构造函数的时候 新构造出来的对象他的this就会指向一个空对象 并且他的原型是他的构造函数 就是 function Demo()

这种原型链的关系在控制台里一目了然

所以刚才那一串代码的对象和属性关系就一目了然了
