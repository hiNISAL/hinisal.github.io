原型链学习

js原型链要从原型说起 而这个又离不开对象

每个对象都有对象级别的三个标签


  • proto

  • class

  • extensible


其中这个原型就是指第一个 proto

简单的来看一个小例子

var cat = function(){name: ‘Tom’, legs: 4};

var obj = new cat();

以这里来看 obj其实是继承自cat这个对象的 所以说cat就是obj的原型 obj的proto标签指向的就是cat

 

接着上面的代码

cat.height = 10;
cat.width  = 1000;

我们把猫添加两个属性分别是长宽

obj的原型会指向cat 而cat的原型会指向Object然后Object最终会指向null

这样的就构成了一条链式的结构 所以叫原型链

我们这样访问obj的属性：

obj.legs;
//返回4

他会返回4

但是obj并没有legs属性 我们可以通过obj.hasOwnProperty(“legs”)来判断legs属性是否属于obj对象 发现返回的会是false 说明obj并没有legs这个属性

这就是因为在obj中查找legs这个属性的时候 如果没找到 就会逐级向上层原型去查找 如果直到找到null还没找到的话 就会返回undefined

 

但这这里面有几个坑点要搞清楚

cat.legs = 100;

在上面的基础上 执行这条代码

这个时候我们

obj.legs;
//返回100

这里返回100并不是修改了原型cat上的legs

而是给obj增加了一个legs属性并且赋值100

同理还是可以通过obj.hasOwnProperty(“legs”)来判断legs是否属于obj 这次会返回true

所以说如果要修改原型上的legs属性的值 可以通过

obj.prototype.legs = 999;

这样来修改

但是如果我们执行了 obj.legs = undefined;

再obj.legs; 就会返回undefined

这并不代表obj中或者obj的原型链上没有legs这个属性

同理还是可以用hasOwnProperty活着 legs in obj来判断

我们可以使用 delete legs删除obj下的legs 从而再访问obj.legs就会返回obj原型上的legs属性的值

利用原型链的特性 可以完成对象的继承
